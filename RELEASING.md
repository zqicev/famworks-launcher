# Выпуск релиза и автообновление

Лаунчер ставится через установщик (NSIS, per-user, без прав админа) и
**сам проверяет обновления** при запуске и раз в 30 минут. Обновления берутся
из GitHub Releases репозитория **`zqicev/famworks-builds`** (публичный — токен
в приложении не нужен).

## Как выпустить новую версию

1. **Подними версию** в `package.json` (`version`). Автообновление срабатывает
   только когда в релизе версия **выше** установленной.
   ```
   1.0.0 → 1.0.1
   ```

2. **Создай токен GitHub** (один раз): [github.com/settings/tokens](https://github.com/settings/tokens)
   - Classic token со scope **`public_repo`**, либо fine-grained с доступом
     **Contents: Read and write** к репозиторию `famworks-builds`.
   - Вставь токен в файл **`.env`** (он в `.gitignore`, в репозиторий не попадёт):
     ```
     GH_TOKEN=ghp_твой_токен
     ```

3. **Запусти публикацию** — токен подхватится из `.env` автоматически:
   ```powershell
   npm run release
   ```
   Это соберёт установщик и зальёт в релиз **3 файла**:
   `FamWorks-Launcher-Setup-x.x.x.exe`, `latest.yml`, `*.blockmap`.

4. **Проверь релиз на GitHub** — он должен быть **опубликован** (не Draft).
   Если создался черновик — нажми «Publish release». Черновики автообновление
   не видит.

## Что увидит пользователь

- При запуске лаунчер тихо проверяет обновления.
- Если есть новее — скачивает в фоне, показывает баннер «Загрузка обновления… %».
- Когда загрузил — баннер «Обновление готово» с кнопкой **Перезапустить**.
- Если не нажать — обновление поставится автоматически при следующем закрытии.

## Важно

- **Все 3 файла обязательны** в релизе. Без `latest.yml` обновление не находится.
  Поэтому публикуй только через `npm run release`, не загружай exe вручную.
- **Не подписано сертификатом** — при первой установке Windows SmartScreen
  покажет «Неизвестный издатель» (Подробнее → Выполнить). Обновления после
  установки идут тихо.
- **Версия в `package.json`** должна расти с каждым релизом.
- Локальная сборка без публикации: `npm run dist` (артефакты в `release/`).
